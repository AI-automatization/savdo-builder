## ЧТО ТАКОЕ SAVDO

**Savdo** — конструктор интернет-магазинов для мелких продавцов Узбекистана, которые сейчас продают через Telegram и мессенджеры.

**Проблема:** Продавцы ведут продажи хаотично — фото товаров в Telegram-канале, заказы в личке, учёт в голове.

**Решение:** Каждый продавец получает свой структурированный онлайн-магазин с каталогом, корзиной, оформлением заказа и чатом с покупателем. Ссылку на магазин продавец кидает в Telegram — покупатель открывает в браузере.

**MVP:** Наличная оплата (cash on delivery). Платёжные системы — в следующей фазе.

**Рынок:** Узбекистан, Ташкент. Язык интерфейса: Русский + Узбекский (переключатель).

---

## КТО ПОЛЬЗОВАТЕЛИ

### 1. Покупатель (Buyer)
- Попадает на магазин по Telegram-ссылке
- Чаще всего — мобильный браузер, нередко медленный 3G
- Не хочет регистрироваться — гость по умолчанию
- OTP (SMS) нужен только для доступа к истории заказов
- Главная цель: быстро найти товар → добавить в корзину → оформить заказ
- Если что-то непонятно — кнопка "Написать продавцу в Telegram" всегда рядом

### 2. Продавец (Seller)
- Регистрируется по телефону через OTP
- Проходит онбординг: заполняет магазин, загружает документы, отправляет на проверку
- Управляет товарами, заказами, чатами — через браузер (десктоп приоритет)
- Получает уведомления о заказах и чатах через Telegram-бота
- Не видит аналитику в MVP (только в будущих версиях)

### 3. Администратор (Admin / Superadmin)
- Внутренний сотрудник команды Savdo
- Проверяет и одобряет/отклоняет продавцов и магазины
- Видит все заказы, чаты, логи
- Каждое действие автоматически записывается в аудит-лог
- Отклонение требует обязательного комментария

## ПРИЛОЖЕНИЕ 1: ВИТРИНА (web-buyer)

**Назначение:** Покупатель видит магазин продавца, выбирает товары, оформляет заказ.

**Главный принцип дизайна:**
- Mobile-first. Большинство пользователей — телефон.
- Быстро. Страница должна загружаться даже на 3G.
- Без лишних шагов. Максимум до кассы — 3 клика.
- Всегда есть кнопка "Написать в Telegram" как запасной вариант.

**Страницы:**

| Страница | URL | Что делает |
|----------|-----|------------|
| Витрина магазина | `/store/[slug]` | Лого, название, описание магазина. Сетка товаров с фото, ценой, остатком. Фильтр по категориям. |
| Карточка товара | `/product/[id]` | Фото галерея (до 5 фото). Название, описание, цена. Выбор вариантов (размер, цвет — зависит от товара). Кнопка "В корзину". Кнопка "Задать вопрос продавцу". |
| Корзина | `/cart` | Список товаров с фото. Изменение количества. Итоговая сумма. Кнопка "Оформить заказ". |
| Оформление заказа | `/checkout` | Имя, телефон покупателя. Доставка или самовывоз. Адрес (если доставка). Кнопка "Подтвердить". |
| Заказ | `/orders/[id]` | Статус заказа (ожидает / подтверждён / в пути / доставлен). Состав заказа (снэпшот — неизменяемый). Кнопка "Чат с продавцом". |
| Чат | `/chat/[threadId]` | Сообщения покупателя и продавца. Контекст: фото товара или номер заказа вверху. Поле ввода + кнопка отправить. |
| Вход | `/auth` | Ввод телефона (+998XXXXXXXXX). OTP код. Только для доступа к истории заказов. |

**Компоненты:**
- `ProductCard` — фото, название, цена (перечёркнутая + текущая если скидка), статус наличия
- `VariantSelector` — кнопки-чипы для выбора (Размер: S M L XL | Цвет: ● ● ●)
- `CartItem` — фото + название + вариант + цена + счётчик количества
- `OrderStatusBadge` — цветной статус (ожидает=жёлтый, подтверждён=синий, в пути=оранжевый, доставлен=зелёный, отменён=красный)
- `TelegramFallbackButton` — кнопка с логотипом Telegram "Написать продавцу"
- `OtpInput` — 6 полей для ввода кода

## ПРИЛОЖЕНИЕ 2: ДАШБОРД ПРОДАВЦА (web-seller)

**Назначение:** Продавец управляет своим магазином — товары, заказы, чаты, настройки.

**Главный принцип дизайна:**
- Операционная скорость важнее красоты.
- Десктоп приоритет, но работает на планшете.
- Быстрое подтверждение заказов (минимум кликов).
- Статусы и счётчики на видном месте.

**Страницы:**

| Страница | URL | Что делает |
|----------|-----|------------|
| Вход | `/login` | Телефон + OTP |
| Онбординг | `/onboarding` | Пошаговый визард: профиль → магазин → документы → Telegram → отправить на проверку |
| Дашборд | `/dashboard` | Статус верификации (баннер). Новые заказы. Непрочитанные сообщения. Быстрые ссылки. |
| Товары | `/products` | Таблица/сетка товаров. Фильтр: все / активные / архив / скрытые. Кнопка "Добавить товар". |
| Добавить/редактировать товар | `/products/new`, `/products/[id]/edit` | Форма с разделами: Основная информация → Медиа (загрузка фото) → Цена → Варианты и остатки → Категории |
| Категории | `/categories` | CRUD список категорий магазина. Drag-to-reorder. Max 20 категорий. |
| Заказы | `/orders` | Список заказов с фильтрами по статусу. Кнопки быстрого действия (подтвердить / отправить). |
| Детали заказа | `/orders/[id]` | Данные покупателя. Состав заказа. Тип доставки. Таймлайн статусов. Кнопка чата. |
| Чаты | `/chats` | Список всех тредов (от товаров и заказов). Счётчики непрочитанных. |
| Чат | `/chats/[threadId]` | Сообщения. Контекст товара или заказа вверху. |
| Настройки магазина | `/store/settings` | Логотип, обложка, название, slug, описание. Контакты (Telegram, Instagram, телефон). Настройки доставки. Переключатель "Магазин открыт". |
| Профиль | `/profile` | Личные данные. Уведомления Telegram. |

**Компоненты:**
- `OnboardingStep` — индикатор шагов (1→2→3→4→5) с галочками
- `VerificationBanner` — баннер статуса (на проверке / одобрено / отклонено с причиной)
- `ProductForm` — многосекционная форма (accordion или tabs)
- `VariantBuilder` — интерфейс добавления групп опций и их значений + цены/остатки
- `MediaUpload` — drag-drop зона + превью загруженных фото (max 5, первое = главное)
- `OrderStatusActions` — кнопки-действия в зависимости от текущего статуса заказа
- `ChatThread` — сообщения с аватарами, временными метками, статусом прочтения
- `PublishToggle` — переключатель "Магазин открыт / Закрыт" с подтверждением

## ПРИЛОЖЕНИЕ 3: ПАНЕЛЬ АДМИНИСТРАТОРА (admin)

**Назначение:** Внутренняя команда Savdo управляет верификацией продавцов, модерацией магазинов/товаров, следит за системой.

**Главный принцип дизайна:**
- Плотные таблицы с данными. Информация важнее эстетики.
- Деструктивные действия (бан, отклонение) — всегда подтверждение + обязательный комментарий.
- Каждое действие оставляет след в аудит-логе.
- Фильтры и поиск на каждой странице со списком.

**Страницы:**

| Страница | Что делает |
|----------|------------|
| `/login` | Отдельный вход для админов |
| `/dashboard` | Очередь модерации: N продавцов ждут / N магазинов ждут. Ссылки на разделы. |
| `/sellers` | Таблица продавцов: имя, телефон, статус верификации, дата регистрации, заблокирован/нет. Фильтры. |
| `/sellers/[id]` | Полный профиль: данные, загруженные документы, связанный магазин, история модерации. Кнопки: Одобрить / Отклонить / Заблокировать. |
| `/stores` | Таблица магазинов: название, slug, продавец, статус, публичный/нет. Фильтры. |
| `/stores/[id]` | Профиль магазина: данные, список товаров, история модерации. Кнопки: Одобрить / Отклонить / Приостановить / Снять приостановку. |
| `/products` | Таблица товаров: название, магазин, статус, видимость. Кнопки: Скрыть / Восстановить. |
| `/orders` | Все заказы платформы. Фильтры по статусу, магазину, дате. |
| `/moderation` | Очередь: продавцы и магазины на проверке. Быстрые карточки с действиями. |
| `/audit-logs` | Таблица всех действий администраторов. Только чтение. |

**Компоненты:**
- `ModerationCard` — карточка в очереди: аватар + имя + дата + кнопки действий
- `ActionModal` — модал подтверждения деструктивного действия с обязательным полем "Комментарий"
- `StatusBadge` — цветной бейдж для статусов (продавца, магазина, товара, заказа)
- `AuditLogRow` — строка лога: кто / что сделал / с чем / когда
- `DocumentViewer` — просмотр загруженных документов продавца (паспорт и т.д.)
- `DataTable` — переиспользуемая таблица с сортировкой, фильтрами, пагинацией

## ДАННЫЕ И МОДЕЛИ (для понимания форм и карточек)

### Товар (Product)
```
- Название (RU/UZ)
- Описание
- Фотографии (до 5, первое = главное)
- Базовая цена (в сумах, UZS)
- Старая цена (если скидка — перечёркнутая)
- Статус: черновик / активный / архив
- Категория (глобальная или магазина)
- Варианты (Размер × Цвет = уникальные SKU с отдельной ценой и остатком)
```

### Заказ (Order)
```
- Номер заказа
- Состав: товары со снэпшотом (цена/название на момент заказа — неизменяемо)
- Покупатель: имя, телефон
- Тип доставки: доставка / самовывоз
- Адрес доставки (если доставка)
- Статус: ожидает → подтверждён → в обработке → отправлен → доставлен / отменён
- Итоговая сумма
```

### Продавец (Seller)
```
- Полное имя
- Тип: физическое / юридическое лицо
- Телефон
- Telegram username
- Статус верификации: не верифицирован / на проверке / верифицирован / приостановлен
- Документы (паспорт / бизнес-документы)
```

### Магазин (Store)
```
- Название
- Slug (URL-имя, неизменяемое после одобрения)
- Описание
- Логотип + обложка
- Статус: черновик / на проверке / одобрен / отклонён / приостановлен / архив
- Публичный: да / нет (отдельный переключатель)
- Контакты: Telegram, телефон, Instagram и др.
- Настройки доставки: бесплатно / фиксированная цена / ручная
```

```
Это не просто стеклянные карточки — это живые, текучие поверхности.
```

## КРИТИЧЕСКИЕ ПРАВИЛА БИЗНЕС-ЛОГИКИ (влияют на UI)

| Правило | Что значит для UI |
|---------|-------------------|
| Один продавец = один магазин | Нет кнопки "Создать ещё магазин" |
| Slug магазина неизменяем после одобрения | Поле slug заблокировано для редактирования |
| Состав заказа неизменяем | Нет кнопки "Редактировать заказ" после создания |
| Отмена заказа только из статусов "ожидает" или "подтверждён" | Кнопка отмены только для этих статусов |
| Корзина — только из одного магазина | При добавлении товара из другого магазина — предупреждение |
| Отклонение требует комментария | Форма отклонения: поле комментария обязательное |
| Каждое действие администратора — в аудит-лог | Автоматически, не видно пользователю |
| Магазин видит покупателям только в статусе "одобрен" + is_public=true | Двойной переключатель в настройках |
