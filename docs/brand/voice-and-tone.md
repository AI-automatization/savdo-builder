# maxsavdo — Voice & Tone

**Версия:** 1.0 · **Дата:** 2026-05-20

> Бренд звучит как **тёплый retail-продавец**, не как **холодный SaaS** и не как **маркетплейс-крикун**. Этот файл — практическое руководство «как мы пишем».

---

## 1. Принципы — четыре правила

### 1.1 Партнёр, не учитель

Мы — рядом с продавцом, не сверху него. Не «нужно сделать», а «можно сделать». Не «вы должны», а «давайте попробуем».

| Плохо | Хорошо |
|-------|--------|
| Вы должны загрузить минимум 1 фото | Загрузите фото — без него товар плохо видно |
| Сначала заполните обязательные поля | Несколько полей ещё ждут вашего внимания |
| Запрещено использовать буквы латинского алфавита для slug | Slug — только английскими буквами (например, `lola-shop`) |

### 1.2 Конкретно, не размыто

| Плохо | Хорошо |
|-------|--------|
| Что-то пошло не так | Не удалось загрузить фото — попробуйте ещё раз |
| Произошла ошибка | Сервер не ответил за 10 секунд. Проверьте интернет и попробуйте снова |
| Операция выполнена успешно | Товар «Платье шёлковое» добавлен в магазин |

### 1.3 Тёплый retail, не SaaS-холод

| Плохо | Хорошо |
|-------|--------|
| Store provisioned successfully | Ваш магазин готов — поделитесь ссылкой |
| Authorization required | Войдите через Telegram, чтобы продолжить |
| Inventory updated | Количество обновлено |

### 1.4 Без маркетплейс-крика

| Плохо | Хорошо |
|-------|--------|
| 🔥 МЕГА-СКИДКА! ТОЛЬКО СЕГОДНЯ! 🔥 | Скидка 20% до конца недели |
| ⚡ HOT DEAL ⚡ | Новая коллекция уже в магазине |
| ❗❗❗ ПОСЛЕДНИЙ ТОВАР ❗❗❗ | Остался последний — спешите |

## 2. Стилистические правила

### 2.1 Обращение

- **Buyer (на витрине):** «Вы» / «Siz». Тёплое, уважительное.
- **Seller (в dashboard):** «Вы» / «Siz», но допустимо для CTA — императив «Создайте магазин» вместо «Создать магазин». Императив звучит как приглашение, не приказ.
- **Push / уведомления:** короткие, императив для action, утверждение для info.

### 2.2 Эмодзи

- **Макс 1 эмодзи на сообщение.** Желательно — 0.
- **Тематический эмодзи OK** в push'ах: 📦 (заказ), ⭐ (отзыв), 🎁 (подарок/промо).
- **Никогда:** 🔥💯⚡🚀💥 и подобные «cryptobro»-эмодзи. И никогда не больше одного на строку.

### 2.3 Капс-лок

- **Капс-лок запрещён** для целых слов длиннее 4 букв в обычном копи.
- **Допустимо** только в overline-метках (см. typography.md): `— ПЕРЕЙТИ В МАГАЗИН`, и то с трекингом.
- **Никогда** для целых заголовков, никогда для error-сообщений, никогда в push'ах.

### 2.4 Восклицания

- **Макс одно `!`** на сообщение, и только когда действительно радостное событие.
- **Никаких `!!!`**, никогда.
- Чаще точка `.` — она звучит спокойнее и взросллее.

### 2.5 Числа и валюта

- **Сум всегда:** `1 250 000 сум` (пробел разделитель, сум — слово, после числа).
- **Никогда:** `1,250,000 UZS`, `$152`, `1250000 sum`.
- **Когда форма «сум» неуместна** (графики, инфографики, бухгалтерия) — `1 250 000 UZS`. Английская валютная аббревиатура — только в техконтексте.

### 2.6 Дата и время

- **Дата:** `15 мая` (короткая), `15 мая 2026` (полная). Никогда `15.05.2026` в UI копи (только в technical-export).
- **Время:** `14:30` (24-часовой). Никогда AM/PM.
- **Период:** `до конца недели`, `сегодня до 18:00`, `на следующей неделе` — естественные.

### 2.7 Узбекская латиница — все `ʻ` правильные

- **`ʻ`** (U+02BB) обязательно: Oʻzbekiston, soʻm, foʻq, oʻquv
- **`ʼ`** (U+02BC) в boʼlim и подобных: используется реже, может опускаться
- **Никогда** `'` (ASCII U+0027) или `'` (smart quote U+2019)

## 3. Десять примеров — «как мы пишем»

### Push #1 — новый заказ продавцу

```
ru: 🛒 Новый заказ №142 от Aziza. 2 товара на 320 000 сум
uz: 🛒 Yangi buyurtma №142 — Aziza. 2 ta tovar, 320 000 soʻm
```

**Почему хорошо:** конкретно (номер, имя, сумма, количество), один эмодзи, нет восклицаний, не SaaS-холод.

### Push #2 — клиент написал в TG

```
ru: Aziza написала в Telegram — откройте чат
uz: Aziza Telegram orqali yozdi — chatni oching
```

**Почему хорошо:** причина действия + CTA в одной строке, no marketing-fluff.

### Success #1 — магазин создан

```
ru: Магазин готов. Поделитесь ссылкой: maxsavdo.uz/lola-shop
uz: Doʻkoningiz tayyor. Havolani ulashing: maxsavdo.uz/lola-shop
```

**Почему хорошо:** констатация + следующий шаг, тёплое слово «готов» вместо «provisioned».

### Success #2 — товар добавлен

```
ru: «Платье шёлковое» добавлено. Посмотреть в магазине?
uz: «Ipak koʻylak» qoʻshildi. Doʻkonda koʻrasizmi?
```

**Почему хорошо:** называет конкретный товар, предлагает следующий step.

### Error #1 — нет интернета

```
ru: Не получилось — проверьте интернет и попробуйте ещё раз
uz: Boʻlmadi — internetni tekshirib, qaytadan urinib koʻring
```

**Почему хорошо:** не пугающий «error», констатация + actionable next step.

### Error #2 — товар закончился

```
ru: «Платье шёлковое, размер S» закончился пока вы оформляли. Попросить продавца уведомить когда появится?
uz: «Ipak koʻylak, S oʻlcham» buyurtma jarayonida tugadi. Sotuvchiga xabar berishni soʻrasinmi?
```

**Почему хорошо:** объясняет ЧТО произошло и КОГДА, предлагает решение (subscribe to restock).

### Error #3 — slug занят

```
ru: Адрес `lola-shop` уже занят. Попробуйте `lola-shop-tashkent` или придумайте свой
uz: `lola-shop` manzili band. `lola-shop-tashkent` yoki oʻzingiznikini taklif qiling
```

**Почему хорошо:** проблема + конкретное решение + альтернатива.

### Empty state — корзина пуста

```
ru: Пока пусто. Откройте каталог и выберите что-нибудь
uz: Hozircha boʻsh. Katalogni oching va biror narsa tanlang
```

**Почему хорошо:** не «нет товаров» (звучит как ошибка), а «пока пусто» — мягкое + invitation.

### Empty state — нет заказов (seller)

```
ru: Заказов пока нет. Поделитесь ссылкой на магазин в Telegram-канале — клиенты придут
uz: Buyurtmalar hali yoʻq. Telegram-kanalingizda doʻkon havolasini ulashing — mijozlar keladi
```

**Почему хорошо:** не упрёк («у вас 0 заказов»), а guidance — что сделать чтобы заказы пошли.

### Marketing — главная страница (hero)

```
ru:
  — ВАШ МАГАЗИН В TELEGRAM
  За минуту — настоящая витрина для тысяч клиентов

  [Создать магазин]   [Посмотреть пример]

uz:
  — TELEGRAM'DAGI DOʻKONINGIZ
  Bir daqiqada — minglab mijozlar uchun haqiqiy doʻkon

  [Doʻkon yaratish]   [Namuna koʻrish]
```

**Почему хорошо:** overline + value-prop + 2 CTAs (primary + secondary). Никакого «революции», «#1 платформы» — простое и конкретное.

## 4. Десять примеров — «как мы НЕ пишем»

### Не-пример #1 — SaaS-холод

```
❌ Your store has been provisioned. Your subdomain is now live.
✅ Магазин готов. Адрес: maxsavdo.uz/lola-shop
```

**Почему плохо:** «provisioned», «subdomain», «is now live» — лексика DevOps, не продавца.

### Не-пример #2 — маркетплейс-крик

```
❌ 🔥 МЕГА АКЦИЯ! -50% ТОЛЬКО ДО ПОЛУНОЧИ! НЕ УПУСТИ! 🔥🔥🔥
✅ Скидка 50% на летнюю коллекцию. До 31 мая
```

**Почему плохо:** капс, множество эмодзи, истерика, no-fact (что именно акция? как именно скидка?).

### Не-пример #3 — корпоративный анонимный

```
❌ В связи с проведением технических работ функциональность платформы может быть ограничена
✅ Завтра с 02:00 до 04:00 ночи возможны короткие перебои. Извините за неудобства
```

**Почему плохо:** «в связи с проведением», «функциональность платформы» — речь чиновника. Не сказано КОГДА и НАСКОЛЬКО.

### Не-пример #4 — извинения-пустышка

```
❌ Извините за неудобства, мы работаем над исправлением
✅ Сейчас не получается загрузить фото. Мы разбираемся — попробуйте через 5 минут
```

**Почему плохо:** generic apology + zero info. Хороший вариант называет ЧТО сломано и КОГДА попробовать.

### Не-пример #5 — двусмысленная ошибка

```
❌ Что-то пошло не так. Попробуйте позже
✅ Сервер перегружен. Подождите минуту и попробуйте снова
```

**Почему плохо:** «что-то» — нулевая инфа. Если знаем что — называем.

### Не-пример #6 — техническое error-сообщение

```
❌ ECONNREFUSED 127.0.0.1:5432
✅ Не удалось связаться с сервером. Проверьте интернет
```

**Почему плохо:** stack-trace в UI — никогда. Перевести в человеческое.

### Не-пример #7 — guilt-trip

```
❌ Вы ещё не загрузили фото — без него ваш магазин не будет продаваться
✅ Загрузите фото — товар с фото покупают в 5 раз чаще
```

**Почему плохо:** «не будет продаваться» — угроза. Хороший вариант — статистика + benefit.

### Не-пример #8 — формально и сухо

```
❌ Регистрация прошла успешно. Ваш аккаунт активирован
✅ Вы вошли. Давайте создадим первый магазин
```

**Почему плохо:** «прошла успешно», «аккаунт активирован» — формуляр госуслуг. Хороший вариант приглашает к следующему шагу.

### Не-пример #9 — feature-jargon

```
❌ Включите push-уведомления, чтобы получать инсайты в реальном времени
✅ Включите уведомления — мы пришлём в Telegram, когда придёт заказ
```

**Почему плохо:** «инсайты в реальном времени» — SaaS-buzzword. Хороший вариант называет КАНАЛ и СОБЫТИЕ.

### Не-пример #10 — «мы» как корпорация

```
❌ Компания maxsavdo поздравляет вас с первым заказом!
✅ Поздравляем с первым заказом! Желаем тысячи следующих
```

**Почему плохо:** «компания maxsavdo поздравляет» — формальный пресс-релиз. Тон должен быть как поздравление от знакомого.

## 5. Двуязычные шаблоны — копипаст-готовые

### 5.1 Push — новый заказ

```
ru: 🛒 Новый заказ №{N} от {имя}. {кол-во} товара на {сумма} сум
uz: 🛒 Yangi buyurtma №{N} — {имя}. {кол-во} ta tovar, {сумма} soʻm
```

### 5.2 Push — клиент написал

```
ru: {имя} написал(а) в Telegram — откройте чат
uz: {имя} Telegram orqali yozdi — chatni oching
```

### 5.3 Email — заказ подтверждён (buyer)

```
ru:
Subject: Заказ №{N} принят — maxsavdo.uz/{shop}
Body:
Здравствуйте, {имя}.
Ваш заказ №{N} на {сумма} сум принят. Продавец напишет вам в Telegram в ближайшее время.

Что заказано:
{список}

Доставка: {адрес}
Контакт продавца: {tg-username}

Спасибо что выбрали {shop}.
— Команда maxsavdo

uz:
Subject: Buyurtma №{N} qabul qilindi — maxsavdo.uz/{shop}
Body:
Salom, {имя}.
{сумма} soʻmlik №{N}-buyurtmangiz qabul qilindi. Sotuvchi tez orada Telegram orqali yozadi.

Buyurtma tarkibi:
{список}

Yetkazib berish: {адрес}
Sotuvchi: {tg-username}

{shop}'ni tanlaganingiz uchun rahmat.
— maxsavdo jamoasi
```

### 5.4 Email — OTP-код

```
ru:
Subject: Ваш код maxsavdo: {code}
Body:
{code} — код для входа.
Не делитесь им ни с кем. Код действует 5 минут.

uz:
Subject: maxsavdo kodingiz: {code}
Body:
{code} — kirish kodi.
Hech kim bilan ulashmang. Kod 5 daqiqa amal qiladi.
```

### 5.5 Bot — приветствие @savdo_builderBOT

```
ru:
Привет! Я бот maxsavdo. Через меня вы:
• создадите магазин за 1 минуту
• будете получать заказы прямо в Telegram
• подтвердите свой номер (без SMS)

Нажмите [Создать магазин] чтобы начать.

uz:
Salom! Men maxsavdo botiman. Men orqali siz:
• 1 daqiqada doʻkon yaratasiz
• buyurtmalarni toʻgʻri Telegram'ga olasiz
• telefon raqamingizni tasdiqlaysiz (SMSsiz)

Boshlash uchun [Doʻkon yaratish] tugmasini bosing.
```

### 5.6 OG-card / share text

```
ru: {shop_name} на maxsavdo — каталог, цены, доставка по Ташкенту
uz: {shop_name} — maxsavdo'da. Katalog, narxlar, Toshkent boʻylab yetkazib berish
```

## 6. Tone matrix — когда какой тон

| Ситуация | Тон | Пример (ru) |
|----------|-----|-------------|
| Onboarding-приветствие | Тёплый, приглашающий | «Создадим вам магазин — это займёт минуту» |
| Success (заказ создан) | Радостный, без истерики | «Заказ принят. Продавец напишет в Telegram» |
| Soft error (re-try possible) | Спокойный, actionable | «Не получилось. Проверьте интернет и попробуйте ещё раз» |
| Hard error (нужна помощь) | Серьёзный, направляющий | «Что-то сломалось у нас. Напишите в @savdo_support — поможем» |
| Empty state | Мягкий, направляющий | «Пока пусто. Откройте каталог и выберите что-нибудь» |
| Marketing (главная) | Уверенный, конкретный | «За минуту — настоящая витрина» |
| Payment / money | Точный, нейтральный | «К оплате: 320 000 сум. Click | Payme | при доставке» |
| Push (notification) | Краткий, конкретный | «🛒 Новый заказ №142 — 320 000 сум» |
| Legal (terms / privacy) | Формальный, чёткий | Юридический язык, но без архаизмов |

## 7. Запрещённые формулировки

| Не использовать | Использовать |
|-----------------|--------------|
| MEGA, СУПЕР, ОГОНЬ, БОМБА | названия товара / события без оценочной риторики |
| Революция, прорыв | конкретная фича |
| Лучший / номер один | конкретное преимущество |
| Бесплатно навсегда (если не точно) | «без комиссии в MVP» / «бесплатно до запуска тарифов» |
| Уведомление получено, операция совершена | конкретное действие («Заказ создан», «Товар сохранён») |
| Что-то пошло не так | конкретная причина |
| Производственный режим, прод | продакшн / реальная среда (если уж надо) |
| Юзер, юзеры | покупатель, клиент, продавец |
| Юзкейс, флоу, фича (в UI копи; в технической документации — OK) | сценарий, действие, возможность |
| Юзабилити (в UI) | удобство |

## 8. Чек-лист для копи

Перед публикацией любого UI-текста / email / push:

- [ ] Звучит как человек, не как машина / документ
- [ ] Конкретно (есть имя, число, событие)
- [ ] Нет капс-локков для слов > 4 букв
- [ ] Макс 1 эмодзи (или 0)
- [ ] Макс 1 восклицание (или 0)
- [ ] Узбекский с правильным `ʻ` (не `'`)
- [ ] Сум через пробел: `320 000 сум`
- [ ] Дата: `15 мая`, не `15.05.2026`
- [ ] Не SaaS-jargon (provisioned, инсайты, флоу)
- [ ] Не маркетплейс-крик (МЕГА, 🔥, !!!)
- [ ] Не угрозы / guilt-trip
- [ ] Если ошибка — называет ЧТО и предлагает ЧТО ДЕЛАТЬ
