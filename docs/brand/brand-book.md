# Savdo — Brand Book (Master)

**Версия:** 1.0 · **Дата:** 2026-05-20

> Этот документ — стратегическое ядро бренда. Все остальные файлы в `docs/brand/` — техническая декомпозиция этих принципов.

---

## 1. Brand mission

> **Savdo помогает продавцам Узбекистана построить настоящий магазин в Telegram за время, которое раньше уходило на регистрацию в маркетплейсе. Покупателю — открыть, посмотреть, написать, купить — без аккаунтов и приложений.**

Мы убираем три барьера UZ-e-commerce:

1. **Сложность маркетплейсов** для продавца — Uzum/OLX требуют документов, недель верификации, комиссий 15-25%.
2. **Холод SaaS** — конструкторы магазинов (Shopify, Tilda) — иностранные, без RU/UZ, без интеграции с Telegram, без сум-валюты.
3. **Пятый аккаунт** — покупатель не хочет регистрироваться. У него уже есть Telegram — этого достаточно.

---

## 2. Positioning

### One-liner

> **«Telegram → витрина → заказ. За минуту.»**

### Полная позиция

Savdo — это не маркетплейс. Это **инструмент для продавца**, который превращает его Telegram-канал в полноценную витрину `savdo.uz/<slug>` с корзиной, каталогом, вариантами и заказами. Продавец остаётся хозяином своих клиентов: ссылка ведёт на его магазин, заказы приходят ему в Telegram, никто не диктует комиссии.

### Чем мы НЕ являемся

- Не маркетплейс (нет общего каталога, нет конкуренции между продавцами на одной странице)
- Не CRM-комбайн (только то, что нужно для продаж: товары, заказы, простая аналитика)
- Не «WhatsApp-Business для UZ» (мы Telegram-native, и это сознательно)
- Не «ещё один Shopify» (мы под UZ-реалии: сум, ʻ-латиница, узб. дни, Click/Payme, доставка по Ташкенту)

### Конкурентная карта

| | Savdo | Uzum Market | OLX Uzbekistan | Shopify / Tilda |
|---|---|---|---|---|
| Время до первой витрины | 1-3 мин | 5-14 дней (верификация) | 5 мин (объявление) | 30-60 мин (настройка) |
| Комиссия | 0% MVP | 15-25% | бесплатно (но без витрины) | $29+/мес |
| Локаль UZ + сум-валюта | ✅ native | ✅ | ✅ | ❌ |
| Telegram-витрина | ✅ TMA + web | ❌ | ❌ | ❌ |
| Продавец = хозяин клиентов | ✅ | ❌ (Uzum видит) | ✅ | ✅ |
| Витрина как сайт | ✅ | ❌ (страница в маркете) | ❌ (объявления) | ✅ |

## 3. Audience — две стороны платформы

### 3.1 Seller (B2B) — «Лола, продавец»

**Профиль:**
- Женщина / мужчина, 25-40 лет, Ташкент / Самарканд / Бухара / Наманган.
- Продаёт через Telegram-канал 200-3000 подписчиков. Категории: одежда (women's wear), косметика, детское, handmade, продукты.
- 2-5 лет онлайн-опыта, но «программистам не дозвонишься».
- Принимает заказы в DM, ведёт учёт в notes / Excel / Google Sheets / голове.
- 1-3 заказа в день в стартовый период, 10-30 в активный.

**Pain points:**
1. Клиенты пишут — она не успевает отвечать вручную, теряет 20-30% заказов.
2. Нет страницы, на которую можно дать ссылку клиенту, кроме TG-канала с тысячами постов.
3. Маркетплейсы съедают маржу и забирают клиентов себе.
4. Не знает английского — Shopify, Wix не подходят.

**Trigger to use Savdo:**
- «Прислал клиент ссылку на каталог конкурента — а у меня просто пост в канале»
- Запуск нового сезона / коллекции
- После 3-го пропущенного заказа в DM

**Что она ценит:**
- Скорость настройки (закинула фото — есть витрина)
- Простоту языка (всё на ru/uz, никаких «инвентаризация / SKU / ESKU»)
- Контроль (видит свои заказы, общается напрямую с покупателем)
- Доверие к платформе (узнаваемый UZ-бренд, не «очередной американский тул»)

**Tone-of-voice для seller:**
- Партнёр, не учитель.
- «Ваш магазин» (не «ваша инстанция платформы»).
- Краткие, конкретные действия. Не SaaS-английский.

### 3.2 Buyer (B2C) — «Азиза, покупатель»

**Профиль:**
- Женщина / мужчина, 18-45 лет, любой регион UZ.
- В Telegram сидит 2-6 часов в день.
- Видит у друга в канале/чате ссылку → переходит в витрину.
- Платит наличными при доставке (60-70%), Click/Payme (20-30%), карта (редко).
- Не хочет ставить ещё одно приложение. Не хочет регистрироваться.

**Pain points:**
1. Каналы Telegram — нет каталога, не понятно что есть, цены устарели, нужно листать вверх-вниз.
2. На сайтах-конкурентах требуют регистрацию ради одной футболки.
3. Цены в долларах / рублях / без указания — не понятно итоговое в сумах с доставкой.

**Trigger to buy:**
- Друг прислал ссылку
- Увидел в чате
- Канал, на который подписан, выложил пост со ссылкой на витрину

**Что он ценит:**
- Быстро открывается (даже на 3G)
- Цены в сумах, понятно сразу
- Можно посмотреть и **написать продавцу в TG** — настоящему человеку
- Не нужно регистрироваться чтобы посмотреть
- OTP только когда хочет историю заказов

**Tone-of-voice для buyer:**
- Тёплый, как продавец на базаре.
- Без капс-локков и crypto-кнопок «CLAIM NOW».
- Уважительный к языку: «Вы / Siz», но не сухой.

## 4. Brand pillars (четыре опоры)

### 4.1 Trust (доверие)

**Что значит:**
- Платформа выглядит и звучит так, как будто за ней стоит реальная UZ-компания. Не «копия Shopify», не «typescript-stack made in spare time».
- Бренд-цвет — terracotta — отсылка к глине, ремеслу, базару — материальным вещам, которым доверяют.
- Контактная инфо везде: «Написать в Telegram», «+998…», номер магазина видно.
- Никогда не обещаем то, что не выполнено («в разработке», «скоро», «beta» — открыто пишем).

**Как проявляется в UI:**
- CTA «Перейти к магазину» — terracotta, плотный, не градиент.
- Status-badges не неоновые, а warm-tinted (success в light = `#4A6B45`, тёплый зелёный).
- Сообщения об ошибках честные («Заказ не оплачен — попробуйте ещё раз», а не «Something went wrong»).

### 4.2 Простота

**Что значит:**
- 60 секунд от регистрации до первой витрины — наш бенчмарк.
- Один экран = одно действие.
- Никаких 10-этапных мастеров настройки. Прислал фото → товар. Прислал номер → магазин.
- Минимум технической лексики. «Адрес магазина» вместо «slug». «Картинка» вместо «media URL». «Цена» вместо «base_price».

**Как проявляется в UI:**
- Spacing scale простая (4/8/12/16/24/32/48/64).
- Кнопок на экране минимум — primary + 1 secondary.
- В seller-dashboard: 4-5 ссылок в sidebar, не 15.

### 4.3 UZ-локальность

**Что значит:**
- Это не «universal e-commerce платформа с UZ-локалью». Это **UZ-first платформа**, которая знает что:
  - Валюта = сум, разделитель тысяч = пробел (1 250 000), без копеек.
  - Узбекская латиница использует `ʻ` (U+02BB) — не `'`, не `'`.
  - Доставка по Ташкенту в день заказа реальна, между регионами — 2-5 дней.
  - Click / Payme / Uzcard — приоритет, Visa/MC — потом.
  - Дни недели и месяцы в обоих языках (ru + uz).
- В UI везде где есть переключение языка — ru / uz одновременно. Узбекская латиница, не кириллица (по решению 2023 года Узбекистан переходит на латиницу).

**Как проявляется в UI:**
- `1 250 000 сум` — не `1,250,000 UZS`, не `$152`.
- Шрифт Inter с subsets `latin + cyrillic` — поддерживает обе письменности.
- Day-pickers, time-zones — Asia/Tashkent по умолчанию.
- В voice-and-tone: «Salom», «Rahmat», «Telegram orqali yozing» — естественные узбекские формы.

### 4.4 Telegram-native

**Что значит:**
- Telegram — не «один из каналов», а **первичная точка контакта** с обеих сторон.
- Seller получает уведомления о заказах в Telegram-боте `@savdo_builderBOT`.
- OTP — только через Telegram-бот, никогда SMS.
- Buyer пишет продавцу в Telegram (не в чат на сайте).
- Витрина TMA (Telegram Mini App) — главный mobile-канал.

**Как проявляется в UI:**
- CTA «Написать в Telegram» — всегда виден на витрине, в `--color-telegram` (`#2AABEE` / `#38BDF8`).
- На главной маркетинговой странице первый action = «Войти через Telegram», не email/password.
- В onboarding продавца — TG-аккаунт собирается сразу, до телефона.

## 5. Tone of voice (краткий обзор)

Полные примеры — в [`voice-and-tone.md`](./voice-and-tone.md).

### Ключевая формула

> **Тёплый retail. Не SaaS-холодный. Не маркетплейс-неоновый.**

### Что значит

- **Тёплый:** «Ваш магазин готов» — лучше, чем «Store provisioned successfully».
- **Retail:** «Заказ принят, продавец напишет в Telegram» — звучит как продавщица в магазине, не как push из банка.
- **Не SaaS-холодный:** избегаем «инстанция / провизионинг / DAU / CTR / churn».
- **Не маркетплейс-неоновый:** избегаем «🔥 МЕГА АКЦИЯ! 🔥», «-90% TODAY ONLY», капслока, многоэтажных эмодзи в копи. Эмодзи — точечно (1 на сообщение макс).

### Двуязычие

- Ru-копия и uz-копия пишутся **параллельно одним человеком**, не «ru написал, в uz перевели через Google».
- Узбекская латиница со всеми `ʻ`: «Oʻzbekiston», «foʻq», «zoʻr».
- Не калькируем русский синтаксис в узбекский. Узбекский → SOV-порядок, естественный для языка.

**Примеры:**
- ru: «Ваш заказ №142 принят. Продавец напишет вам в Telegram.»
- uz: «Buyurtmangiz №142 qabul qilindi. Sotuvchi sizga Telegram orqali yozadi.»

## 6. Brand visual essence

(Полная техдекомпозиция — в `palette.md`, `typography.md`, `logo-spec.md`, `components-guide.md`.)

### 6.1 Цвет

**Primary:** terracotta `#7C3F2E` (light) / `#A05A45` (dark).
**Surface:** кремовый `#FBF7F0` (light) / warm near-black `#16120D` (dark).
**Запрещено:** generic SaaS-violet (`#7C3AED` остаётся только как technical accent в seller-CRM, никогда в brand-точках).

### 6.2 Типографика

**Шрифт:** Inter (`latin + cyrillic`) с feature settings `cv11 ss01`.
**Wordmark:** «Savdo» — Inter Semibold, всегда capital S, никогда в курсиве, никогда с эффектами.

### 6.3 Логотип

- Wordmark «Savdo» — основной носитель.
- Monogram «S» — для favicon, app icon, маленьких размеров (< 32px).
- Иконка ShoppingCart рядом с wordmark — допустимо для контекстов где нужна семантика «магазин» (footer marketing-сайта). На витрине buyer/seller — wordmark одиночный.

### 6.4 Tactile, не glossy

- **Нет** градиентов на CTA.
- **Нет** glassmorphism в основном UI (исключение: onboarding-orbs seller'а — единственная допустимая точка glass).
- **Нет** drop-shadow выше уровня 16px (`shadow-lg` — потолок).
- **Да** — плоские solid-color поверхности с warm-tone границами.

## 7. Когда и как применять

### 7.1 Применяется к

- `apps/web-buyer/**` — материнская точка
- `apps/web-seller/**` — подтянуть к buyer (см. пункт 7.2)
- `apps/tma/**` — Telegram Mini App, использует Telegram theme + наши tokens где это возможно
- `apps/admin/**` — internal CRM, может отступить (но wordmark + brand-цвет логотипа = соответствует)
- `apps/api/**` — email-шаблоны, push-нотификации, OG-карточки, bot-сообщения → соответствуют `voice-and-tone.md`
- Marketing-сайт (если будет отдельно) — соответствует полностью
- Печатные материалы, визитки, банеры — соответствуют

### 7.2 Зоны ответственности (из CLAUDE.md)

| Зона | Кто | Что делает с brand book |
|------|-----|------------------------|
| `apps/web-buyer` | Азим | Реализует — это эталон |
| `apps/web-seller` | Азим | Подтягивает к buyer (см. 5 wins в audit) |
| `apps/tma` | Полат | Использует tokens из brand book где возможно |
| `apps/admin` | Полат | Минимум: wordmark + brand-цвет логотипа |
| `apps/api` (email/push/bot) | Полат | Использует `voice-and-tone.md` |
| Marketing / Telegram-каналы | — (не определено) | Полное соответствие |

### 7.3 Когда отступать

Допустимо отступать, **но не игнорировать**, в следующих случаях:

1. **Telegram WebApp (TMA):** Telegram даёт свои переменные темы (`tg.themeParams.bg_color` и т.д.) — их можно использовать поверх наших tokens. Wordmark — наш.
2. **Per-store branding (планируется):** продавец загружает свой логотип/цвет — он перекрывает CTA витрины. Wordmark `Savdo` в footer/credit остаётся нашим.
3. **A/B-тесты:** можно тестировать вариации CTA-копи, размеров кнопок, hero-layout. Brand-цвет, шрифт, wordmark — не тестируем.
4. **Internal admin:** Полат может использовать минимально brand book (только wordmark + цвет в логотипе). Внутренние таблицы — функциональность над брендом.

### 7.4 Что нельзя никогда

- **Не растягивать wordmark.** Только uniform scale.
- **Не накладывать эффекты на logo** (тени, обводки, свечения, gradients).
- **Не использовать violet `#7C3AED` как brand.** Только как dashboard accent в seller-CRM.
- **Не использовать generic green / red shadcn-цвета для semantic.** Только warm-tinted (см. `palette.md`).
- **Не использовать `'` или `'` вместо `ʻ`** в узбекской латинице.
- **Не использовать «MEGA», «🔥🔥🔥», капслок** в копи.

## 8. Чек-лист «бренд-консистентный экран»

Перед запуском любого нового экрана / страницы / письма / push'а пройди:

- [ ] Wordmark «Savdo» — в `colors.brand`, Inter Semibold, capital S
- [ ] Primary CTA — `colors.brand` background + `colors.brandTextOnBg`
- [ ] Фон — `colors.bg` (`#FBF7F0` light / `#16120D` dark)
- [ ] Текст body — `colors.textBody` (не `text-primary` для длинных параграфов)
- [ ] Все границы — `colors.border` (`#EFE8DA` light / `rgba(...,0.12)` dark)
- [ ] Apostrof в uz-копи — `ʻ` U+02BB, не `'`
- [ ] Сумма — `1 250 000 сум`, не `1,250,000 UZS`
- [ ] Эмодзи — макс 1, тематический (📦 для заказа OK; 🔥💯⚡ — нет)
- [ ] Tone — partner, не teacher; warmth, не enthusiasm
- [ ] Telegram CTA — `colors.telegram`, всегда видна на storefront
