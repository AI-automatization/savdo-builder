# 01_product_scope.md

## 1. User Roles

### 1.1 Buyer
Покупатель, который:
- просматривает товары
- добавляет в корзину
- оформляет заказ
- общается с продавцом

### 1.2 Seller
Продавец, который:
- создаёт магазин
- добавляет товары
- управляет заказами
- общается с клиентами

### 1.3 Admin (Superadmin)
Полный контроль над системой:
- пользователи
- магазины
- товары
- заказы
- чат
- модерация


---

## 2. Buyer Flow

1. Заходит по ссылке магазина
2. Просматривает товары
3. Открывает карточку товара
4. Выбирает вариант (размер, цвет и т.д.)
5. Добавляет в корзину
6. Переходит в checkout
7. Вводит:
   - имя
   - телефон
   - адрес
   - комментарий
   - тип получения (delivery / pickup)
8. Подтверждает заказ
9. Получает доступ к чату


---

## 3. Seller Flow

1. Регистрация
2. Подтверждение телефона
3. Создание магазина
4. Заполнение профиля
5. Добавление товаров
6. Отправка на модерацию
7. Публикация магазина
8. Получение заказов
9. Общение с покупателями


---

## 4. Store Model

- у каждого продавца свой магазин
- магазин доступен по ссылке
- у магазина нет общей конкуренции с другими (в MVP)
- продавец сам приводит трафик


---

## 5. Cart & Orders

- одна корзина = один продавец
- нельзя смешивать товары разных магазинов
- заказ создаётся внутри платформы
- статус заказа управляется продавцом


### Order Status (MVP)

- pending
- confirmed
- cancelled
- completed


---

## 6. Checkout Model

Поддерживаемые методы:

- Cash on Delivery (основной)
- Manual payment (по договорённости)
- Online payment (в будущем)

---

## 7. Chat Scope

Поддерживается:

- чат по товару (до заказа)
- чат по заказу (после)

Функции:
- текст
- изображения
- уведомления

Не поддерживается:
- полноценный мессенджер
- группы
- звонки


---

## 8. Product Model

Товар включает:

- название
- описание
- цена
- старая цена (optional)
- фото (1+)
- категории
- варианты:
  - размер
  - цвет
  - другие

Каждый вариант:
- SKU
- stock
- цена (optional override)


---

## 9. Categories

- глобальные категории (platform-level)
- локальные категории продавца (store-level)


---

## 10. Seller Verification

Этап 1:
- регистрация
- создание магазина

Этап 2:
- подтверждение телефона
- Telegram
- данные продавца
- модерация

Только после этого:
- магазин становится публичным


---

## 11. Notifications

Поддержка:

- mobile push
- web push

События:
- новый заказ
- сообщение
- изменение статуса заказа


---

## 12. Admin Scope

Admin может:

- банить пользователей
- удалять товары
- управлять заказами
- читать чаты
- модерировать магазины
- управлять системой полностью


---

## 13. What is NOT included in MVP

- полноценный маркетплейс
- сложная логистика
- автоматические возвраты
- сложная платёжная система
- рейтинги и отзывы
- аналитика продавца
