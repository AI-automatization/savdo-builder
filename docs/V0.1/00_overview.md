# 00_overview.md

## 1. Product Overview

Название: TBD (working: Savdo / StoreGo / Shop Builder)

Продукт представляет собой e-commerce платформу для продавцов, которые сейчас продают товары через Telegram и другие мессенджеры.

Платформа предоставляет:
- индивидуальный магазин (storefront) для каждого продавца
- удобный каталог товаров
- оформление заказа внутри приложения
- встроенный чат buyer ↔ seller
- альтернативный канал связи через Telegram

Основная цель — заменить хаотичную торговлю через чаты на структурированную систему продаж.


---

## 2. Problem Statement

На текущий момент в Узбекистане большинство малых продавцов используют Telegram для продаж.

Основные проблемы:
- отсутствует структура товаров (всё в виде постов)
- сложный поиск и навигация
- нет корзины и нормального checkout
- отсутствие доверия к продавцу
- коммуникация смешана с контентом
- отсутствие системной обработки заказов

Покупатель:
- вынужден скроллить каналы
- не может удобно сравнивать товары
- не имеет прозрачного процесса покупки

Продавец:
- вручную обрабатывает заказы
- теряет клиентов
- не имеет инструментов управления


---

## 3. Solution

Создание платформы, которая:

Для продавца:
- даёт собственный онлайн-магазин
- упрощает управление товарами
- систематизирует заказы
- добавляет чат с клиентом

Для покупателя:
- даёт удобный интерфейс выбора товаров
- корзину и checkout
- прозрачный процесс покупки
- возможность общения с продавцом

Платформа не заменяет Telegram полностью, а дополняет его.


---

## 4. Product Positioning

Это НЕ классический маркетплейс (как Uzum / Wildberries).

Это:
> инфраструктура для продавцов (Shop Builder + Store Platform)

Ключевое отличие:
- продавец приводит свой трафик
- у каждого свой магазин
- нет общего “рынка” на старте

Фокус:
- малые и средние продавцы
- Telegram-first бизнес


---

## 5. Target Audience

### Primary
- малые продавцы (одежда, обувь, косметика, техника)
- Telegram магазины
- Instagram магазины

### Secondary
- покупатели этих продавцов


---

## 6. Core Features (MVP)

- регистрация продавца
- создание магазина
- добавление товаров (с вариантами)
- витрина магазина
- корзина (1 seller = 1 cart)
- checkout
- заказы
- чат buyer ↔ seller
- push уведомления
- Telegram контакт + уведомления

---

## 7. Business Model (Future)

На этапе MVP монетизация отключена.

Планируемая модель:
- подписка для продавцов
- дополнительные функции (analytics, branding, продвижение)


---

## 8. Success Metrics (V0.1 MVP)

- 30 активных продавцов
- 100 опубликованных товаров
- 20 опубликованных магазинов
- 10 завершённых заказов
- 40% retention продавцов (7 дней)


---

## 9. Key Constraints

- один заказ = один продавец
- основной рынок: Узбекистан
- основной способ оплаты: наличные при получении
- online payment — опционально (не критично для MVP)
- логистика не автоматизируется


---

## 10. Long-term Vision

Платформа становится:
- стандартом для малых продавцов
- инфраструктурой e-commerce вне маркетплейсов
- инструментом масштабирования бизнеса

Возможное развитие:
- встроенные платежи
- реклама внутри платформы
- аналитика
- маркетплейс-слой поверх магазинов
- интеграции (CRM, ERP)
