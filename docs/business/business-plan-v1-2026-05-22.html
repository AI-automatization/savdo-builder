1. Аудитория

Платформа двухсторонняя: B2B (продавцы платят) и B2C (покупатели бесплатно).

🛍 B2B — продавцы (paying side)

Telegram-channel seller ~15-30k в UZ
Малый бизнес уже продаёт в TG-канале (одежда, food, crafts). Канал 1-50k подписчиков.
😩 Корзина и заказы — вручную в DM. Заказы теряются. Нет аналитики.
Instagram-микро-продавец ~20-40k в UZ
Продаёт через сторис/DM. Хочет упорядочить flow и не терять заказы.
😩 IG не оптимален для commerce, всё руками, нет checkout.
Региональный продавец ~10-20k в UZ
Не-Tashkent: Самарканд, Бухара, Фергана, регионы. Локальная аудитория.
😩 Uzum логистика медленная/дорогая в регионах, локальный охват слабый.
Self-employed / ремесленник ~5-10k в UZ
Керамика Риштана, сюзани, атлас, food-artisan (мясо-молочка, выпечка).
😩 Аутентика не вписывается в массовые маркетплейсы; нужен свой storefront.

🛒 B2C — покупатели (free side)

TG-активный молодой UZ ~8-10M в UZ
18-35 лет, 80% времени в Telegram, открыт к новым платформам.
Региональный покупатель ~10-12M в UZ
Uzum-доставка слабая/дорогая, доверяет локальным продавцам.
Loyal buyer конкретного канала ~3-5M в UZ
Уже покупал у продавца в DM, хочет normal checkout без торгов.
НЕ наша аудитория —
Покупатели бытовой электроники (Uzum/Asaxiy); cross-border (AliExpress); большие brands (свои сайты).

3. Что внутри платформы — Feature Map

Три приложения: storefront для покупателя (web + TMA), dashboard для продавца (TMA + web-seller), admin-панель (Полат + Азим).

🛒 Buyer-side (web-buyer + TMA buyer)

Discovery

  • Главная с топ-магазинами и категориями
  • Каталог магазинов с фильтрами и сортировкой
  • Каталог товаров платформа-wide
  • Поиск по магазинам/товарам
  • Категории первого уровня (тренды UZ)
  • Персонализированная feed (recommendation)

Storefront

  • Per-seller storefront по slug-URL
  • Cover image, лого, описание магазина
  • Список товаров с вариантами
  • Trust signals: verified badge, rating, reviews count
  • Direct link для шейринга в TG
  • Custom storefront templates (Pro+)

Product detail

  • Галерея фото (multi-image)
  • Цена + sale (со скидкой —NN%)
  • Варианты (размер, цвет, etc) с отдельным stock
  • Описание, характеристики (attributes)
  • Отзывы с рейтингом
  • Stock indicator (есть/мало/нет)
  • Wishlist (heart icon)
  • «Из этого магазина» — кросс-продажи

Cart & Checkout

  • Корзина = один store (INV-C01)
  • Quantity controls, удалить
  • Cross-store warning (новый store → reset)
  • Sync между TMA и web-buyer (cart sync API)
  • Checkout с OTP-gate (Telegram-auth)
  • Delivery mode: доставка / самовывоз
  • Address autocomplete (Yandex Suggest)
  • Payment method: cash / card (disabled / online (Phase 2)
  • Idempotent confirm (защита от двойных заказов)

Post-purchase

  • История заказов (статусы PENDING → DELIVERED)
  • Order detail с timeline статусов
  • Chat buyer↔seller на заказ
  • Push-нотификация в Telegram при смене статуса
  • Wishlist с notify (price drop, back-in-stock)
  • Reviews после доставки
  • Возвраты/refunds

Profile

  • Telegram-привязанный аккаунт
  • Имя, аватар (синк с TG)
  • Сохранённые адреса доставки
  • Уведомления в TG + in-app
  • Языковой переключатель ru/uz
  • Login через Telegram OTP
  • CTA «Стать продавцом» (upgrade flow)

🏪 Seller-side (web-seller + TMA seller)

Dashboard

  • Метрики: продажи за период, заказы, GMV
  • Графики (last 30 days)
  • Топ товаров
  • Recent orders (последние 10)
  • Уведомления о новых заказах (WS + TG)
  • Quick actions: добавить товар, новый магазин

Каталог товаров

  • CRUD товаров (добавить/изменить/удалить)
  • Multi-image upload + crop
  • Variants matrix (size × color × material)
  • Stock per variant
  • Attributes (динамические по категории)
  • Category filters (мульти-выбор)
  • Sale price (со скидкой)
  • Status: DRAFT / ACTIVE / ARCHIVED
  • Bulk операции (hide/show, archive)

Заказы

  • Список заказов с фильтрами по статусу
  • Order detail с buyer info + items
  • State machine: PENDING → CONFIRMED → PROCESSING → SHIPPED → DELIVERED
  • «Взять в обработку» / отмена
  • Chat с покупателем
  • Stock auto-deduction при создании заказа
  • Stock restoration при отмене

Магазин

  • Название, slug-URL, описание
  • Logo + cover image
  • Категории магазина (внутренние)
  • Контакты: TG-username, телефон
  • Настройки доставки: fixed / manual / none
  • Поддерживаемые регионы доставки
  • Pickup-настройки
  • Статус публикации (DRAFT → APPROVED → PUBLISHED)

Chat

  • Buyer-to-seller threads с привязкой к заказу/товару
  • Text + photo messages
  • Read/unread indicators
  • WebSocket real-time
  • Edit (15 min window) + delete (soft)
  • Report message (модерация)
  • TG-нотификация о новом сообщении

Подписка & биллинг

  • Просмотр текущего тарифа
  • История платежей
  • Upgrade / downgrade тарифа
  • Annual discount применение
  • Реферальный код, статистика приглашений
  • White-label (Business+)

⚙️ Admin / Platform Ops (наш control panel)

Управление

  • Пользователи (поиск, suspend, impersonate via JWT)
  • Продавцы (verify/unverify, история)
  • Магазины (approve, suspend, archive)
  • Товары (hide, restore, delete)
  • Manual seller activation (POS/ERP flow)
  • Admin users (super_admin / admin / moderator)

Модерация

  • Очередь модерации с SLA-таймером
  • Action history (approve/reject/comment)
  • Detail page по кейсу
  • Reject требует комментария (INV-A02)
  • Audit log всех admin actions (INV-A01)
  • Reports (жалобы на сообщения)

Operations

  • Dashboard с key metrics платформы
  • Analytics: events, summary
  • System health, feature flags
  • Broadcast (Telegram-рассылка)
  • DB CRUD (super_admin only)
  • Media migration tools (TG→R2/Supabase)
  • Audit log viewer + filters
  • Audit broken media URLs
