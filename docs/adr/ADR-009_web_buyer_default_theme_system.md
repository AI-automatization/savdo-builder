# ADR-009 — Default theme web-buyer = `system`

**Дата:** 2026-05-30 (Азим)
**Статус:** Accepted
**Зона:** `apps/web-buyer` (buyer storefront). web-seller / admin — отдельно.

## Контекст

Закрывает `BRAND-DARK-VS-LIGHT-DEFAULT-001`. После ребрендинга на maxsavdo
(ADR-008) и палитры **Dark Luxury Minimalist** встал вопрос: какой default-темы
ставить новому посетителю web-buyer, у кого ещё нет выбора в localStorage.

Текущая реализация (`apps/web-buyer/src/app/layout.tsx`): `ThemeScript` +
`ThemeProvider` оба с `defaultTheme="system"`. Переключатель темы
(`ThemeToggle`) присутствует и сохраняет выбор в `localStorage['savdo-theme']`
— при любом решении пользователь может переключиться вручную.

Рассмотренные варианты:
- **A. `dark`** — brand-forward, каждый видит Dark Luxury.
- **B. `system`** (текущий) — следуем OS-настройке.
- **C. `light`** — классический e-com, лучшая читаемость фото.

## Решение

**Оставляем `system`.** Кода не меняем — это осознанное решение, а не недоделка.

### Обоснование

web-buyer — это **buyer-facing storefront, метрика №1 — конверсия**, не
brand-identity. Аргументы против форсирования `dark`:

1. **UGC-фото товаров.** Фото от мелких продавцов (одежда/еда/крафт) часто на
   белом/ярком фоне. На тёмном фоне → жёсткие «вырезы», просадка читаемости.
   Эффект неравномерный и нами не контролируется (зависит от съёмки продавца).
2. **Контекст рынка UZ — светлый.** Uzum / Olcha / Asaxiy все light. Покупатель
   из Telegram-ссылки ожидает привычный яркий commerce; тёмный storefront
   незнакомого бренда = трение в момент первого впечатления.
3. **Нет аналитики конверсии.** Sentry frontend ещё не подключён — ставить
   «brand-forward, но рискованно» вслепую, без возможности померить и откатить,
   не стоит.

Почему `system`, а не `light`:
- `system` не жертвует брендом — кто в OS-dark (заметная доля молодой
  TG-аудитории) увидит Dark Luxury. Мы не навязываем и не гадаем за пользователя.

Где бренд выражается без риска для конверсии:
- Золотой accent (Champagne Gold), логотип, типографика Inter — работают в обеих темах.
- Полноценный Dark Luxury — на **seller-dashboard** (там default уже `dark`,
  CRM-identity), маркетинг-страницах, Telegram-боте.

## Последствия

- **Изменений в коде нет** — `defaultTheme="system"` остаётся в обоих местах.
- Существующих пользователей с выбором в localStorage решение не затрагивает.
- **Пересмотреть, если:** появится аналитика конверсии (Sentry frontend +
  событийный трекинг) и команда захочет A/B `dark` vs `system` с мониторингом и
  откатом. Тогда — отдельный тикет.
