# ADR-002 — Один магазин на продавца в MVP

**Дата:** 2026-03-21
**Статус:** Accepted

## Контекст

Нужно определить: сколько магазинов может создать один продавец?

## Решение

В MVP **один seller = один store**.

Технически: `stores.seller_id UNIQUE`.

## Причины

- Упрощает checkout flow — нет вопроса "из какого магазина этот заказ"
- Упрощает seller dashboard — нет переключения между магазинами
- Упрощает модерацию — один объект на проверку
- Целевая аудитория (Telegram-продавцы) обычно имеет один "бренд"
- Снижает риск создания фейковых аккаунтов

## Последствия

- Constraint: `UNIQUE(seller_id)` в таблице stores
- Бизнес-логика: при попытке создать второй магазин — `STORE_ALREADY_EXISTS`
- Схема уже готова к снятию этого ограничения в V1.0+ (seller_id не composite PK)

## Что изменится в V1.0+

При снятии ограничения нужно:
- Убрать UNIQUE constraint
- Добавить переключатель магазинов в seller dashboard
- Обновить checkout flow для выбора магазина
