# CLAUDE.md — orchestrator protocol

> Прочитай этот файл **в начале каждой сессии** в savdo-builder.

## Протокол сессии

### 1. Начало сессии — войти в курс дел

```
1. ОТКРЫТЬ orchestrator/inbox.md
   → если есть необработанные записи (без отметки [PROCESSED]) — прочитать,
     учесть в плане сессии. Если есть срочное — обработать первым.

2. ОТКРЫТЬ orchestrator/dashboard.md
   → восстановить контекст: активные задачи, статус, открытые вопросы.

3. ОТКРЫТЬ orchestrator/alerts.md
   → если не пуст — упомянуть пользователю в первом ответе сессии.

4. (Не обязательно) orchestrator/sessions.md — что было в прошлой сессии.
```

### 2. Во время сессии — обновлять

- Новая идея/задача обнаружена → добавить в `inbox.md` (сверху, с timestamp).
- Что-то требует решения пользователя → добавить в `alerts.md`.
- Не трогать `inbox.md`-записи, помеченные `[PROCESSED]` — это для истории.

### 3. Конец сессии — закрепить состояние

```
1. ОБНОВИТЬ dashboard.md
   → актуальные активные задачи, прогресс, что следующее.

2. ОТМЕТИТЬ обработанные записи в inbox.md как [PROCESSED 2026-05-21 19:30 by claude:session-N]
   → не удалять, только пометить.

3. ДОБАВИТЬ запись в sessions.md (append, сверху)
   → дата, что сделано, что осталось, ссылки на коммиты.

4. ПРОВЕРИТЬ alerts.md — если что-то закрыто, удалить.
```

## Формат записей

### inbox.md
```markdown
## 2026-05-21 18:45 [Полат]
Текст события. Может быть многострочный.
Контекст важен — пиши как для другого человека.
```

После обработки claude добавляет:
```markdown
## 2026-05-21 18:45 [Полат] [PROCESSED 2026-05-21 19:30 by claude:session-N]
...
```

### dashboard.md
Не append, а **полная перезапись** в конце сессии. Структура:
```markdown
# Dashboard — состояние мира

**Обновлено:** 2026-05-21 19:30 (claude:session-N)

## 🔴 Активные задачи
1. ...
2. ...

## ⏳ Ждут (внешняя зависимость)
- ...

## ❓ Открытые вопросы для Полата
- ...

## ✅ Закрыто в последней сессии
- ...

## 🎯 Следующее действие
...
```

### alerts.md
Короткие пункты с action item:
```markdown
- [ ] 2026-05-21 19:00 — TMA build всё ещё падает. Проверь Settings → Builder.
- [ ] 2026-05-21 19:15 — Sentry DSN не задан, observability не активна.
```
Закрытые пункты удалять (история есть в sessions.md).

### sessions.md
Append-only, newest at top:
```markdown
## Session 2026-05-21 18:00 → 19:30 (claude:opus-4-7)

**Сделано:**
- ...

**Обнаружено:**
- ...

**Осталось:**
- ...

**Коммиты:** abc1234, def5678
```

## Правила

- **Не быть многословным** в этих файлах — это рабочая память, не литература.
- **Не дублировать** `analiz/done.md`, `analiz/tasks.md`, `analiz/logs.md` — orchestrator высокоуровневый, эти — детальный трекинг.
- **Если в начале сессии нет inbox-записей и dashboard пустой** — нормально, значит между сессиями ничего не произошло.
- **`[PROCESSED]` обязательно** — иначе следующая сессия обработает повторно.

## Не трогать в orchestrator/

- `README.md` — это для Полата, не для меня (Claude не должен его «улучшать» каждую сессию).
- `CLAUDE.md` (этот файл) — менять только если структура реально изменилась, не косметически.
