# Orchestrator — async-канал между Полатом и Claude

Stage 1 (созданО 2026-05-21). Только файлы, никаких процессов на ноутбуке.

## Идея

Между сессиями Claude не существует — он живёт только в момент, когда ты пишешь. Но накопленная между сессиями память может быть. Эта папка — общая «доска».

Если у тебя возникла идея, задача, вопрос, баг — не теряй, пока ждёшь следующей сессии. Запиши в `inbox.md`. При следующем `pnpm dev:api` (или просто открытии Claude Code в этой папке) я **первым делом** читаю `inbox.md` и `dashboard.md`, чтобы войти в курс дел.

Это не делает меня непрерывным. Это делает наш канал асинхронным — ты пишешь когда удобно тебе, я обрабатываю когда сессия открыта.

## Структура

| Файл | Кто пишет | Кто читает | Назначение |
|------|-----------|-----------|-----------|
| `inbox.md` | Полат (+ позже Telegram-бот) | Claude в начале сессии | Очередь событий: идеи, задачи, баги, наблюдения. Append-only. Newest at top. |
| `dashboard.md` | Claude в конце сессии | Полат в любой момент | Текущая картина мира: активные задачи, прогресс, открытые вопросы. |
| `alerts.md` | Claude когда нашёл проблему | Полат, Telegram-бот (Stage 2) | Что **требует** твоего внимания. Не информация — действие. |
| `sessions.md` | Claude в конце каждой сессии | Полат, ретроспектива | Лог: когда сессия, что сделано, что осталось. Append-only. |
| `CLAUDE.md` | один раз | Claude в начале сессии | Протокол: как взаимодействовать с этой папкой. |

## Как пользоваться (для Полата)

### Записать событие
Открой `inbox.md`, добавь блок **сверху**:
```markdown
## 2026-05-21 18:45 [Полат]
Мысль: надо проверить что Sentry правильно скраббит phone в production-логах.
Не сейчас, как дойдём.
```
Сохрани. При следующей сессии Claude увидит.

### Посмотреть статус
Открой `dashboard.md`. Там текущая картина.

### Если что-то требует тебя
Смотри `alerts.md`. Если пустой — всё под контролем.

### Откатить всё
`rm -rf savdo-builder/orchestrator/` — папка удалена, никаких хвостов на машине нет.

## Что Stage 1 даёт (и чего НЕ даёт)

✅ Не теряются мысли между сессиями
✅ Claude входит в сессию с контекстом, не «с нуля»
✅ Видна история — что сделано, что осталось, почему
✅ Никаких процессов на ноутбуке, никаких токенов, никаких новых установок

❌ Claude **не** проактивен — пока ты не начал сессию, ничего не происходит
❌ Нет push-уведомлений — Stage 2 (Telegram-бот) добавит
❌ Нет периодического мониторинга — Stage 3 (cron-Claude) добавит, если решим

## Следующие stage'ы (если Stage 1 себя оправдает)

- **Stage 2:** Telegram-бот пишет в `inbox.md` твои сообщения. Когда `alerts.md` обновляется — бот тебе пушит.
- **Stage 3:** cron-Claude просыпается каждые N часов, читает inbox, обновляет dashboard. Здесь начинается реальная стоимость API.
- **Stage 4:** sub-agents для конкретных задач — мониторить Railway, scan Sentry, отслеживать ветки.

Решаем по итогам Stage 1 (минимум неделя живого использования).
