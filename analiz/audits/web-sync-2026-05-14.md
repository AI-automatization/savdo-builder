## Часть 1 — Sync OK

Что работает правильно (audit явно подтвердил):

- **API endpoint hygiene** — 100% web-* запросов попадают в существующие controllers с canonical paths (`/api/v1/{buyer,seller,storefront}/*`) и правильными методами/role-guards. 0 dead, 0 stale, 0 duplicate.
- **Token refresh logic** — идентичная в web-buyer и web-seller, без infinite loops.
- **OTP** — ТОЛЬКО Telegram bot (`@savdo_builderBOT`). 0 references на Eskiz / Playmobile / Twilio / AWS SNS / Nexmo / SMS. Web-buyer OtpGate, web-seller login, TMA, admin — все используют `/auth/otp`.
- **INV-C01 (cart=один store)** — web-buyer не имеет multi-store cart UI или hook'а; backend enforce.
- **INV-C03 (заказ immutable)** — нет UI для редактирования order items; только cancel + создать новый.
- **INV-S01 (один seller=один store)** — web-seller не имеет route'а или UI для создания второго store.
- **User roles** — только BUYER / SELLER / ADMIN / HYBRID. 0 unauthorized roles (MODERATOR/VENDOR/RESELLER).
- **Стейт-машина статусов заказа** — web-buyer canCancel = `[PENDING, CONFIRMED]`, web-seller CANCELLABLE = `[PENDING, CONFIRMED, PROCESSING]`, NEXT_TRANSITION map совпадает с `docs/V1.1/02_state_machines.md`.
- **Stock / sale price / order status / delivery fee** — всегда из API, не вычисляется на frontend (`API-DELIVERY-FEE-CLIENT-CONTROLLED-001` resolved 10.05.2026).
- **Media URL resolution** — 100% backend (`product-presenter.service.ts:resolveImageUrl`). Frontend читает `images[].url` / `mediaUrls[]` напрямую. Нет hardcoded supabase URLs, нет `bucket`/`objectKey` references, нет локальных `resolveMediaUrl()` функций.
- **Env vars** — `NEXT_PUBLIC_TG_BOT_USERNAME` только для deep-links/OTP gate. Нет `*_TG_BOT_URL` для media.
- **Type imports** — web-* импортирует `Product`, `ProductListItem`, `Cart`, `Order`, `ChatThread`, `CheckoutPreview`, `AddCartItemRequest` из `packages/types`. Нет локальных `interface Order`, `interface Cart` etc.
- **Refund flow** — admin-only (нет refund UI в web-buyer / web-seller).
- **Chat deep-link** — `chat_<threadId>` consistent: TMA `HomePage.tsx`, Bot `telegram-notification.processor.ts:170`.
- **Store share link** — `store_<slug>` consistent: TMA `StorePage.tsx:58`, `ProfilePage.tsx:59`.
- **No gamification / loyalty / referral / affiliate** features.
- **No admin-level features** inside seller dashboard.

### Pillar 5 — Ideology / scope-creep

**🔴 SEV-1 — INV-CH02 violation: chat edit/delete shipped без ADR**
- File: `apps/web-buyer/src/lib/api/chat.api.ts:49-56`, `apps/web-seller/src/lib/api/chat.api.ts:51-58`
- Backend: `apps/api/src/modules/chat/chat.controller.ts:260-303`
- Что: `DELETE /chat/threads/:id/messages/:msgId` + `PATCH /chat/threads/:id/messages/:msgId` реализованы end-to-end (UI + API + backend).
- INV-CH02: «Удаление и редактирование сообщений не поддерживаются в MVP».
- **Контекст (Азим знает):** edit/delete для chat был задизайнен и вшит в session 36 (26.04.2026) Полатом + мной. Это сознательное расширение, но **ADR в `docs/adr/` отсутствует**. Без ADR это выглядит как scope creep.
- Action: либо (a) написать ADR-XXX «relax INV-CH02: chat messages can be edited/deleted by author within N minutes, soft-delete via `is_deleted` flag», либо (b) откатить (но это уже в проде используется).

**🔴 SEV-1 — `card` payment option misleading в checkout**
- File: `apps/web-buyer/src/app/(minimal)/checkout/page.tsx:272-277` (option), :383 (confirm), :394 (analytics)
- Что: payment option `id: 'card'` имеет `disabled: false` и labelled "Картой курьеру / UzCard / Humo POS-терминал". Buyer может выбрать. **НО** `paymentMethod` selected state НИКОГДА не отправляется в API — `confirm.mutateAsync` (line 383) не передаёт `paymentMethod`, а `CheckoutConfirmRequest` в `packages/types/src/api/cart.ts` НЕ имеет `paymentMethod` поля. Analytics event (line 394) hardcode'ит `"COD"` независимо от выбора.
- Эффект: покупатель видит селектор который ничего не делает; продавец не видит что покупатель выбрал «card».
- Action: либо (a) `disabled: true` с «Скоро» badge до того как Полат добавит `paymentMethod` в `CheckoutConfirmRequest` + payment provider integration, либо (b) расширить контракт + UI правильно отправляет field.

**🟡 SEV-3 — Product reviews UI undocumented**
