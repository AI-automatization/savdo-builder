## P1 — Серьёзные (нарушают ключевую функциональность)

### P1-1 [Магазины] Нет UI-поля для Telegram-канала магазина
- **Где:** `apps/admin/src/pages/StoreDetailPage.tsx` (или эквивалент)
- **Что:** функция `postProductToChannel` определена в коде apps/api, но **поле `channelId` (Telegram-канал) не редактируется через UI Admin Panel**. Продавец/админ не может задать @channel_username для магазина → товары при публикации **никогда не попадают в Telegram-канал** автоматически.
- **Импакт:** мёртвая ключевая фича marketplace-promotion. Каждый магазин в проде сейчас без `channelId`.
- **Фикс (Полат):** добавить input в форму магазина в admin + endpoint PATCH /admin/stores/:id для обновления `telegramChannelId`.

### P1-2 [Аналитика] Несоответствие: 0 заказов + выручка 6886 сум
- **Где:** `apps/api/src/modules/analytics/` (aggregation queries)
- **Что:** Dashboard Аналитики показывает "0 заказов за 30 дней", при этом "выручка 6 886 сум, средний чек 3 443 сум". Если заказов 0, выручка не может быть >0.
- **Гипотезы:**
  - Разные временные окна в aggregation queries (выручка считается за весь период, заказы за 30 дней)
  - aggregation использует `orders` без фильтра `cancelledAt`, а count — с фильтром
  - Округление при `_sum.totalAmount` не учитывает `cancelled`
- **Фикс:** ревизия `GetAnalyticsUseCase` + унификация фильтров.

### P1-3 [Dashboard] Race condition при первом рефреше
- **Где:** `apps/admin/src/pages/DashboardPage.tsx`
- **Что:** при cold-start счётчики показывают `0/0/0/0` пока API не ответил (~14с). Нет skeleton/loading state — пользователь видит "пустую панель" вместо лоадера.
- **Импакт:** UX, пользователь может подумать что данных нет.
- **Фикс (Полат):** добавить skeleton-loader или `loading=true` state, не рендерить нули до получения данных.

### P1-4 [Товары] Расхождение UI=7 vs DB=26
- **Где:** Admin Panel → Товары vs Admin Panel → База данных → products
- **Что:** список товаров показывает 7 записей, БД через раздел "База данных" показывает 26 строк в таблице products.
- **Гипотезы:**
  - UI фильтрует по `status NOT IN (DRAFT, ARCHIVED)` неявно
  - UI фильтрует по `deletedAt IS NULL` — а в БД raw count
  - Pagination (`limit=20` по умолчанию, и default страница пуста? — маловероятно)
- **Фикс:** ревизия `apps/api/src/modules/admin/repositories/admin-products.repository.ts` (или эквивалент) — какие фильтры применяются по умолчанию.

### P1-5 [TMA] Guest-режим без initData (security concern)
- **Где:** `apps/tma/` (entry point)
- **Что:** TMA открывается через прямой URL (без `Telegram.WebApp.initData`) → показывает **полноценный каталог в guest-режиме**, навигация по магазинам/корзине работает без авторизации.
- **Импакт:** возможный security concern (зависит от того, открытая ли это фича или баг).
- **Решение требует обсуждения:** нужна ли заглушка для guest-доступа (требовать initData), или это сознательное design choice (storefront-доступ публичный, checkout требует OTP)?
