## 🔑 КЛЮЧЕВАЯ ГИПОТЕЗА (Ahmed)

**Практически все ошибки загрузки (403/error) — скорее всего ОДНА root cause:**

> Telegram аккаунт Полата НЕ привязан как seller в БД
> (telegramId либо отсутствует у user, либо user не связан с seller-row)

Если гипотеза верна — **8 из 8 P0/P1 закроются одним фиксом данных** (linkage user.telegramId ↔ seller).

## Проверки следующего шага

1. **Admin Panel → Users → найти Полата** (по phone или другой identifier) — проверить:
   - `telegramId` заполнен?
   - `role` = `SELLER` или просто `BUYER`?

2. **Admin Panel → Sellers** — найти запись с `userId` = Полат:
   - Существует ли seller-row?
   - `verificationStatus` = VERIFIED?
   - `telegramUsername` совпадает с Telegram?

3. **БД (через раздел "База данных") → stores** — найти магазин Полата:
   - `sellerId` указывает на seller-row выше?
   - `status` = APPROVED?

## Если гипотеза подтвердилась

Решение через UI:
- Admin Panel → Sellers → New → создать seller-запись для Полата → привязать к user
- Создать store → подтвердить
- Передеплой не нужен — это data fix

## Если гипотеза НЕ подтвердилась (linkage есть, но всё равно 403)

Тогда **код-баг в auth-цепочке backend**:
- Возможно JWT не парсит seller-context правильно
- Или middleware теряет контекст
- Зона: `apps/api/src/modules/auth/` + `apps/api/src/modules/seller/`
